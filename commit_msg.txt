build(ci): 添加 Build APK 工作流 (4 ABI)

新增 .github/workflows/build-apk.yml:
- 触发: 手动 (workflow_dispatch) / push (app/src/include/cmake/third_party 改动) / PR
- 手动触发可选择 ABI 过滤 (单 ABI 或 4 ABI)
- 多 ABI: arm64-v8a / armeabi-v7a / x86_64 / x86
- 步骤:
  1. checkout (submodules: recursive)
  2. setup-java@v4 (JDK 17 Temurin)
  3. setup-android@v3 (SDK 34, NDK r27d, CMake 3.22.1)
  4. setup-gradle@v4 (Gradle 8.10 + 缓存)
  5. 环境展示 (调试用)
  6. gradle :app:assembleRelease (含 ABI 过滤)
  7. APK 校验 (file + unzip -l)
  8. upload-artifact@v4 (30 天保留)

修改 app/build.gradle.kts:
- abiFilters 加 "x86" (32-bit Intel, 模拟器支持)
- 注释说明 4 个 ABI 的目标场景

工具链说明:
- Gradle 走 NDK 自带 android.toolchain.cmake (不需要我的 custom toolchain)
- 我的 cmake/Options.cmake 用 CMAKE_SYSTEM_NAME / CMAKE_SYSTEM_PROCESSOR
  自动适配所有 4 个 ABI
- SM8635 走 aarch64 + -mcpu=cortex-a75 优化
- 老设备 (32 位) 走 arm + -mfpu=neon -mfloat-abi=softfp
- x86/x86_64 跳过 NEON flag
